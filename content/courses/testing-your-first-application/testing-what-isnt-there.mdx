# Testing what isn't there

Sampai saat ini, kami telah menulis tes dan pernyataan untuk memastikan bahwa segala sesuatunya ditampilkan dengan benar. Misalnya, pengujian **harus menambahkan item baru ke bagian bawah daftar** memastikan bahwa todos kami terdaftar dalam urutan yang benar dan teks penghitungan todo sudah benar.

Ini semua baik dan bagus, tetapi juga penting untuk menulis tes untuk memastikan bahwa segala sesuatunya _not_ ditampilkan ketika tidak seharusnya. Jadi, selain tes kasus positif, pastikan juga tes negatifnya.

Ini akan lebih masuk akal ketika kita menulis tes "negatif" pertama kita. Misalnya, saat todo ditambahkan ke aplikasi, `<footer>` ditambahkan ke bagian bawah aplikasi. Jika tidak ada todos, `<footer>` akan dihapus.

Mari kita tulis tes untuk memastikan bahwa `<footer>` tidak terlihat oleh pengguna kami saat tidak ada todos.

```js
it("does NOT display the footer when there are no todos", () => {})
```

Sekarang mari ambil elemen `<footer>` dan tulis pernyataan bahwa elemen tersebut tidak ada jika tidak ada todos.

```js
cy.get(".footer").should("not.exist")
```

![](/images/testing-your-first-application/testing-what-isnt-there/Screen_Shot_2021-06-25_at_2.33.00_PM.png)

Bagus, ujian kita lulus. Mari kita juga memikirkan beberapa elemen lain yang seharusnya tidak ada ketika tidak ada todos.

Elemen lain yang seharusnya tidak ada adalah elemen **.todo-list** kita. Mari kita tulis pernyataan lain untuk memastikan bahwa ini tidak ada jika tidak ada todos.

```js
cy.get(".todo-list").should("not.exist")
```

Karena kita sekarang memeriksa beberapa elemen, mari perbarui nama pengujian kita menjadi lebih eksplisit.

```js
it("does NOT display the footer or todo-list when there are no todos", () => {
  cy.get(".footer").should("not.exist")
  cy.get(".todo-list").should("not.exist")
})
```
